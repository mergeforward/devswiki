    独立Hello函数到一个文件中

    如果我们将上述文件拆分成main.go和hello.go文件，并放在一个文件夹下，类似

    package main
import "hello"

func main() {
    Hello()
}


    package hello

import "fmt"

// Hello func
func Hello() {
    fmt.Println("hello world")
}

    其会有报错，要么是找不到hello的包（不在gopath下面），要么是一个文件夹下有多个package（main和hello）

    这里我们假设我们使用go mod init basic来初始化，这样当前的目录就相当与GOPATH下的一个目录了，
    另外，考虑到在一个目录下，golang只允许一个package name，我们需要把hello.go放在独立的文件夹下面

    我们在目录结构上做如下的修改

    .
├── go.mod
├── hello
│   └── hello.go
└── main.go

    其中go.mod内容如下

    module basic

go 1.14

    hello/hello.go 内容如下

    package hello

import "fmt"

// Hello func
func Hello() {
    fmt.Println("hello world")
}

    main.go内容如下

    package main

import h "basic/hello"

func main() {
    h.Hello()
}

    这样，当我们使用go run main.go运行依旧可以运行。

    要注意的是，package name和directory name是有些差异的，
    从引入本地包的角度，要使用directory name, 而从安装的角度要使用package name。

    比如当上面的hello文件夹改为了a，那么在main文件中，应将import改为“basic/hello”。
    而basic决定了go install时，生成的执行文件名称为basic。
